# Django REST Framework To-Do API

A robust To-Do list API built with Django REST Framework, featuring JWT cookie authentication, role-based permissions, filtering, pagination, ordering, search, and comprehensive documentation.

## Features ✨

-   **User-Specific To-Do Items:** Every to-do item is securely scoped to the authenticated user.
-   **JWT Cookie Authentication:** Secure, automatic authentication using JSON Web Tokens managed via secure `HttpOnly` cookies.
-   **Comprehensive Authentication Flows:** Leverages `dj-rest-auth` and `django-allauth` for user registration, login, password resets, email verification, and more.
-   **Role-Based Permissions:** A granular permission system ensures data security:
    -   **Superusers:** Full access to all API endpoints and data.
    -   **Staff Users:** Read-only access to users list and details; read-only access to all to-dos; full CRUD access *only* to their own to-do items.
    -   **Authenticated Users:** Can only list, view, create, update, and delete their *own* to-do items.
-   **CRUD Operations:** Standard Create, Read, Update, Delete operations for to-do items, governed by permissions.
-   **Advanced Querying & Data Management:**
    -   **Filtering:** filtering capabilities using `django-filter` (e.g., filter by user, is_done).
    -   **Pagination:** API responses are paginated for efficient handling of large datasets.
    -   **Ordering:** Sort to-do items by various fields (e.g., due date, priority).
    -   **Search:** Integrated search functionality to quickly find specific to-do items.
-   **User Management:** Dedicated `ViewSet` for user-related operations, with appropriate permission checks.
-   **OpenAPI/Swagger UI Documentation:** Interactive API documentation generated by `drf-spectacular` for easy exploration and testing.

## Technologies Used 🛠️

-   **Core:** Python, Django
-   **API Framework:** Django REST Framework (DRF)
-   **Authentication:**
    -   `djangorestframework-simplejwt` (for JWT generation)
    -   `dj-rest-auth` (for auth flows like login, registration, password reset)
    -   `django-allauth` (for user account management)
-   **Documentation:** `drf-spectacular` (for OpenAPI 3 / Swagger UI)
-   **Filtering & Querying:** `django-filter`, DRF built-in Pagination & Ordering

## Setup and Installation 🚀

1.  **Clone the repository:**
```bash
git clone https://github.com/Secondmib69/Todo-API.git
cd Todo-API

2. Create and activate a virtual environment:
bash
python -m venv venv
# On Windows:
# venv\Scripts\activate
# On macOS/Linux:
source venv/bin/activate

3. Install dependencies:
bash
pip install -r requirements.txt

4. Apply database migrations:
bash
python manage.py makemigrations
python manage.py migrate

5. Create a superuser:
bash
python manage.py createsuperuser

*   When prompted, ensure `Is superuser? (yes/no)` is set to `yes`. This user will have full administrative access.
*   Staff users and additional user roles can be managed via the Django Admin or API endpoints.
Run the development server:
bash
6. python manage.py runserver
```
API Endpoints & Documentation 📚
The API is accessible at http://127.0.0.1:8000/api/docs.

Authentication is primarily handled via JWT tokens set in HttpOnly cookies, managed by dj-rest-auth.
